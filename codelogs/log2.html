Code Log 2: CSS Measuring Units

What is the difference between absolute and relative CSS length units?

In CSS, absolute units and relative units are different in how they define size. Absolute units have a fixed and exact size, and they do not depend on other elements or the screen size. Because of this, they remain consistent and do not change automatically.
In contrast, relative units are based on another value, such as a parent element, the root element, or the viewport size. This means their size can change depending on the context.

px, pt

The unit px stands for pixels. It is commonly used in web design and represents a fixed unit on the screen. It is considered an absolute unit because it does not depend on other elements.
The unit pt stands for points and is mostly used in print design. One point equals 1/72 of an inch. While px is more common for screens, pt is more common for printed materials.

vmax, vmin

The unit vmax stands for viewport maximum. It is a relative unit based on the viewport size. 1vmax equals 1% of the larger value between the viewport width and height. It scales according to the larger dimension of the screen.
The unit vmin stands for viewport minimum. It is also a relative unit based on the viewport size. 1vmin equals 1% of the smaller value between the viewport width and height. It scales according to the smaller dimension of the screen.

in, pc

The unit in stands for inches and is an absolute unit based on physical measurement. It is mostly used for print layouts.
The unit pc stands for picas, and 1 pica equals 1/6th of an inch. Like inches, picas are also more common in print design rather than web design. These units are precise but not commonly used for responsive layouts.
